**Aircraft Routing Suite (ARS)** — это высокопроизводительное комплексное веб-приложение для управления и оперативного контроля расписанием полетов авиакомпании.

Система предоставляет единую цифровую среду для работы с рейсами, обеспечивая прозрачность операций, строгий контроль изменений и гранулярное разграничение прав доступа. Архитектура решения построена по принципу **controllers - services - repositories** с использованием современного стека C++ для обеспечения максимальной скорости и надежности.


## Архитектура

Особое внимание в проекте уделено безопасности данных и защите от распространенных векторов атак. Бэкенд реализован без использования тяжелых ORM, что дает полный контроль над SQL-запросами.

### Ключевые механизмы защиты:

* **Защита от инъекций:** Все взаимодействие с базой данных происходит исключительно через **Prepared Statements** (`exec_params` в `libpqxx`). Динамическая конкатенация строк в SQL полностью исключена.
* **Хэширование паролей:** Пароли не хранятся в открытом виде. Используется **SHA-256 + Соль** для каждого пользователя.
* **Ролевая модель доступа:** Разделение на зоны ответственности: `Public` (Табло), `Operator` (Редактирование), `Admin` (Управление пользователями и аудит).
* **Журнал аудита:** Любое изменение данных (создание, правка, удаление) фиксируется в защищенном журнале аудита с привязкой к оператору, времени и конкретным изменениям.


## Бизнес Логика

Система не просто хранит данные, но и обеспечивает их логическую целостность в реальном времени

1.  **Контроль временных петель:** Валидация хронологии (вылет не может быть позже прилета).
2.  **Контроль доступности самолетов:** Система блокирует назначение самолета на рейс, если он находится в полете или на обслуживании в указанный промежуток времени.
3.  **Контроль доступности пилотов:** Проверка занятости пилотов. Невозможно назначить одного пилота (КВС или Second Officer) на пересекающиеся рейсы.


## Стек разработки

### Бэкенд 
* **Язык:** C++17 (STL, Умные указатели, Лямбда функции).
* **Веб Фреймворк:** — [Crow](https://github.com/CrowCpp/Crow), фреймворк для создания HTTP API.
* **Фреймворк базы данных:** `libpqxx` — официальный C++ коннектор для PostgreSQL.
* **Шифрование:** OpenSSL

### База данных
* **СУБД:** PostgreSQL 15.
* **Скрипты:** SQL-скрипты инициализации (`init_schema.sql`) и сидирования данных (`test_data.sql`).

### Инфраструктура
* **Контейнеризация:** Docker и Docker Compose (изолированные среды для DB и App).
* **Сборка:** CMake.

### Фронтенд
* **Стэк:** Vanilla JS, HTML5, CSS3.
* **Дизайн:** Модульная структура, адаптивный интерфейс.


## Установка и запуск

Сборка проекта полностью реализована через Docker. Для запуска требуется только Docker Engine.

```bash
# Клонирование репозитория
git clone https://github.com/kshkiiii/Aircraft-Routing-Suite.git

# Сборка и запуск контейнеров
docker-compose up --build

```
Сервер будет запущен на порту 8080 (http://localhost:8080)

При первом запуске база данных автоматически заполняется тестовым набором данных (30 рейсов).

Администратор: admin / admin (Полный доступ + Аудит)

Оператор: operator / operator (Управление рейсами)
